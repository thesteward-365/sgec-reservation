# 🚀 External Proxy NAS 풀스택 프로젝트 배포 가이드

## 1. 시스템 아키텍처 개요

본 가이드는 SSH 터미널 기반의 설정을 우선하며, GitHub을 통해 이미지 빌드 및 크론 잡(Cron Job)을 관리하는 구조입니다.

---

## 2. 사전 준비 (GitHub)

### 2.1 Next.js 프로젝트 설정

`next.config.js` 파일을 수정하여 Docker 최적화 빌드를 활성화합니다.

```javascript
const nextConfig = {
  output: 'standalone', // 중요: 빌드 결과물을 가볍게 만듦
};
```

### 2.2 CI 워크플로우 구성 (`.github/workflows/deploy.yml`)

코드를 푸시하면 GHCR(GitHub Container Registry)로 이미지를 업로드합니다.

---

## 3. 서버 환경 세팅 (External Proxy SSH)

### 3.1 디렉토리 및 보안 설정

SSH 접속 후 프로젝트 루트 폴더와 환경 변수 파일을 생성합니다.

```bash
# 프로젝트 폴더 생성
mkdir -p /volume1/docker/my-app/postgres_data
cd /volume1/docker/my-app

# 환경 변수 작성 (DB 접속 정보 및 시크릿 키)
vi .env

# GitHub 패키지 접근 권한 인증
# Password 입력란에 GitHub Personal Access Token(PAT) 입력
docker login ghcr.io -u [GITHUB_ID]

```

---

## 4. 서비스 배포 (Docker Compose)

`/volume1/docker/my-app/docker-compose.yml` 파일을 작성합니다.

```yaml
version: '3.8'
services:
  # 1. Database: 외부 노출 없이 내부망 전용으로 설정
  db:
    image: postgres:16-alpine
    container_name: nextjs-db
    environment:
      - POSTGRES_DB=mydb
      - POSTGRES_PASSWORD=${DB_PASSWORD}
    volumes:
      - ./postgres_data:/var/lib/postgresql/data
    restart: always

  # 2. Application: Next.js Standalone 실행
  app:
    image: ghcr.io/[ID]/[REPO]:latest
    container_name: nextjs-app
    env_file: .env
    restart: always
    depends_on:
      - db

  # 3. Watchtower: 5분마다 이미지 갱신 확인 및 알림
  watchtower:
    image: containrrr/watchtower
    container_name: watchtower
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock
      - /root/.docker/config.json:/config.json:ro
    command: --interval 300 --cleanup nextjs-app
    environment:
      - WATCHTOWER_NOTIFICATIONS=discord
      - WATCHTOWER_NOTIFICATION_DISCORD_WEBHOOK_URL=${DISCORD_WEBHOOK}
      - WATCHTOWER_NOTIFICATION_LEVEL=info
    restart: always
```

**배포 실행:**

```bash
docker-compose up -d

```

## 5. 데이터 관리 및 모니터링

### 5.1 DB 백업 스크립트 (`backup.sh`)

데이터 오염 및 실수에 대비해 매일 백업을 수행합니다.

```bash
#!/bin/bash
BACKUP_DIR="./backups"
TIMESTAMP=$(date +%Y%m%d_%H%M%S)
docker exec nextjs-db pg_dump -U [USER] [DB_NAME] > $BACKUP_DIR/backup_$TIMESTAMP.sql
find $BACKUP_DIR -type f -name "*.sql" -mtime +7 -delete

```

- **등록:** 외부 프록시 **작업 스케줄러**에서 해당 스크립트를 매일 실행하도록 등록.

### 5.2 구글 캘린더 동기화 (GitHub Actions Cron)

서버 외부에서 주기적으로 API를 호출하여 동기화 상태를 유지합니다.

- `.github/workflows/sync.yml` 생성
- `schedule: - cron: '*/15 * * * *'` (15분마다 호출)
- **인증:** API 호출 시 헤더에 Bearer 토큰 검증 로직 포함 필수.

### 5.3 Uptime Kuma 상태 페이지

- **설정:** 별도 컨테이너로 띄운 뒤 `Status Page` 생성.
- **보안:** 관리 대시보드는 숨기고, 외부에는 비밀번호가 걸린 상태 페이지 주소만 노출.

---

## 6. 최종 운영 프로세스 요약

1. **개발:** 로컬 작업 후 `git push`.
2. **빌드:** GitHub Actions가 GHCR에 이미지 업로드.
3. **갱신:** Watchtower가 새 이미지를 감지하여 컨테이너 자동 재시작.
4. **동기화:** GitHub Cron이 주기적으로 API를 찔러 구글 캘린더 동기화 수행.
5. **확인:** 디스코드 알림 및 Uptime Kuma 상태 페이지를 통해 정상 작동 확인.

---

## 💡 주요 팁

- **SSH 최소화:** 최초 세팅 이후에는 `git push`와 External Proxy **Container Manager UI**(로그 확인용)만으로 운영 가능합니다.
- **복구:** `postgres_data` 볼륨이 호스트 경로에 매핑되어 있어, 컨테이너를 지우고 다시 띄워도 데이터는 유지됩니다.
