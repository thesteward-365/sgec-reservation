좋다. 지금까지 논의된 방향을 기준으로, 구현 관점에서 빠짐없이 전체 기능을 정리한다. 불필요한 장식 없이 핵심만 명확하게 적는다.

# 1. 서비스 개요

- 목적: 장소와 날짜를 선택하고 시간대를 예약한다
- 제약: 동일 장소/동일 시간 중복 예약 불가
- 연동: 예약 결과를 Google Calendar에 반영
- 사용 패턴: 사전 예약(주 단위, 몇 주 전)

---

# 2. 사용자 기능

## 2.3 예약 상세 (핵심)

역할: 시간 선택 및 예약 생성

- 시간대별 예약 구간 표시
- 예약자 이름 표시

### 빈 시간 구간 추출

- 예약 사이의 사용 가능 시간 계산
- 구간 단위로 표시

### 시간 선택

- 빈 시간 구간 선택
- 혹은 시작-종료시간 선택
- 중간에 겹치는 시간은 선택 불가.
- 연속된 시간을 등록할 수 있음

### 사용 목적 입력

- 직접 입력
- 자주 사용하는 목적 선택

### 예약 처리

- 생성
- 수정
- 취소

## 4.3 캘린더 연동

대상: Google Calendar

### 4.3.1 구조

하나의 구글 계정에 연결하고, 역할이 다른 두 캘린더를 사용

- 내부 캘린더 (Write)
- 외부 캘린더 (Read, 참고용)

### 4.3.2 내부 캘린더 (Write)

- 방향: 우리 → 구글
- 예약 생성/수정/취소 시 이벤트 동기화
- event_id로 1:1 매핑 관리
- 기준 데이터는 우리 시스템

### 4.3.3 외부 캘린더 (Read)

- 방향: 구글 → 우리
- 이벤트 조회 후 화면에 표시 함
- 예약 가능 여부 및 충돌 판단에 사용하지 않음

### 4.3.4 동기화 전략

- 실시간 동기화: 예약 변경 시 즉시 반영
- 주기적 동기화: 누락/실패 복구용 보정 작업

### 4.3.5 동기화 기준

- 우리 시스템을 기준으로 캘린더를 보정
- 불일치 시 우리 데이터로 덮어쓰기

### 4.3.6 관리자 설정

- 구글 계정 연결
- 내부 캘린더 1개 선택
- 외부 캘린더 1개 선택
- 동기화 상태 확인 및 수동 실행
