# Next.js 환경에서 iOS PWA 스플래시 화면 미작동 해결 기록

Next.js 환경에서 아이폰 13 미니를 타깃으로 PWA 스플래시 화면(Startup Image)을 적용하는 과정에서 겪은 문제들과 해결 과정을 기록해 둔다.

## 1. 발생한 문제
- **문제 1 (서브 페이지에서 다운로드 시 404 에러)**: Next.js 레이아웃 메타데이터의 `startupImage` 경로를 상대 경로(`splash_screens/...`)로 작성했다. 메인 페이지(`/`)에서는 작동했지만, 서브 페이지(예: `/admin/reservations` 또는 `/reserve`)에서 앱을 다운로드/설치하여 구동할 때 이미지를 불러오지 못하고 404 에러가 발생했다.
- **문제 2 (스플래시 미작동 및 흰 화면 현상)**: HTML 소스상에 기기별 스플래시 링크 태그들이 다 선언되어 있었음에도 불구하고, 아이폰에서 홈 화면에 추가한 후 실행하면 스플래시 이미지 대신 빈 흰색 화면만 떴다. 웹 검사기로 네트워크 탭을 확인해도 스플래시 이미지 다운로드 요청이 아예 없었다.

## 2. 해결을 위한 디버깅 삽질의 과정
처음에는 이미지 자체의 문제이거나 사파리의 캐시가 원인이라고 판단하여 다음과 같은 시도를 반복했다.
- **캐시 삭제**: 사파리 캐시 지우기, 인터넷 방문 기록 및 웹사이트 데이터 삭제, 사파리 앱 강제 종료 등을 수십 번 시도했다.
- **이미지 포맷 변경**: 이미지 확장자를 JPG에서 PNG로 변경해 보거나, 해상도를 임의로 조정해 보았다.
- **기타 시도**: PWA 아이콘 경로를 수정해 보거나 서비스 워커 파일을 수정하는 등 여러 가지 시도를 했으나 모두 실패했다.

## 3. 원인 분석
결국 원인은 캐시나 파일 포맷이 아니라 **애플 전용 메타 태그의 누락**이었다.
- Next.js의 표준 메타데이터 설정인 `appleWebApp: { capable: true }`를 사용하면 표준 태그인 `<meta name="mobile-web-app-capable" content="yes">`만 렌더링되고, **iOS 사파리가 standalone(독립형) 앱으로 인지하는 데 필요한 필수 태그인 `<meta name="apple-mobile-web-app-capable" content="yes">`가 누락**되는 버그가 있었다.
- 이 태그가 없어서 iOS 사파리는 해당 사이트를 일반 웹사이트로 취급했고, 홈 화면에 추가할 때 스플래시 이미지를 수집 및 캐싱하려는 시도조차 하지 않았던 것이다.

## 4. 해결 방법

### iOS standalone 메타 태그 강제 주입
Next.js의 버그를 우회하기 위해 `metadata.other` 속성을 사용해 헤더에 필수 태그를 수동으로 강제 선언했다.
```typescript
other: {
  'apple-mobile-web-app-capable': 'yes',
},
```

### 스플래시 이미지 절대 경로 수정
서브 페이지 경로에서도 이미지를 정상적으로 로드할 수 있도록 모든 `startupImage` 경로를 절대 경로(`/splash_screens/...`)로 수정했다.
```typescript
url: '/splash_screens/iPhone_13_mini_...'
```

### 스플래시 이미지 자동화 생성 툴 사용
기기별(아이폰, 아이패드 모델 등)로 해상도가 각기 다르기 때문에 스플래시 이미지를 일일이 수작업으로 만드는 것은 매우 비효율적이다. 
이를 위해 오픈소스 자동화 툴인 **`pwa-asset-generator`**를 사용했다.
```bash
# npx를 사용하여 원본 로고 이미지 한 장으로부터 기기별 스플래시 이미지와 미디어 쿼리 태그를 자동 생성했다.
npx pwa-asset-generator logo.png public/splash_screens --background "#ffffff" --scrape false
```
이 툴을 통해 생성된 40여 개의 기기별 이미지 파일과 `<link rel="apple-touch-startup-image" ...>` 태그 리스트를 `layout.tsx`에 그대로 붙여넣어 완벽하게 해결했다.

## 5. 정리
- **요청이 보이지 않는 이유**: 스플래시 이미지는 PWA가 켜질 때 실시간으로 요청되는 것이 아니라, 사파리 브라우저에서 '홈 화면에 추가'를 클릭하는 순간 다운로드되어 기기에 영구 보관된다. 앱 구동 시에는 로컬 캐시를 보여주므로 네트워크 탭에 요청이 보이지 않는 것이 정상이다.
- **원인 정리**: 시차나 파일 포맷, 단순 캐시 문제로 오해하여 많은 시간을 낭비했으나, 핵심은 `<meta name="apple-mobile-web-app-capable" content="yes">` 태그의 유무였다.
